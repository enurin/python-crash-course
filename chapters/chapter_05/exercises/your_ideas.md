# Exercise 05-13: Your Ideas

## Summary of the Book Task

Write down a few original program ideas you could build with conditionals.

## Why This Is Markdown

This exercise is conceptual, exploratory, or depends on a larger project setup
(Pygame, Matplotlib/Plotly data files, web APIs, Django, or deployment). A single
standalone `.py` file would be misleading, so this file records the exact practice
goal in a clean study format.

## Beginner-Friendly Solution Plan

1. Reopen the chapter project files from the book.
2. Make the smallest change that satisfies this exercise.
3. Run the project after the change.
4. Check the visible behavior or terminal output.
5. Commit or save only after the project still works.

## Notes

- Keep the change at the chapter level; do not jump ahead to later techniques.
- If the exercise asks you to read documentation, write down two or three useful
  discoveries in this file.
- If the exercise asks you to extend a project, prefer one clear working feature
  over several half-finished ideas.
